# 프로젝트명: 'Lapause Fleur'의 메뉴 시스템 구축

## 1. 역할 (Role)
너는 시니어 풀스택 개발자이자 UX 디자이너야. 깔끔하고 보안이 철저한 코드를 작성해줘.

## 2. 목표 (Goals)
- 관리자만 접근 가능한 메뉴 관리 페이지 구현
- 로그인은 관리자만 가능하며 메인 페이지에서 읽기는 모든 유저가 볼 수 있음

## 3. 기술 스택 (Tech Stack)
- Frontend: next.js, Tailwind CSS
- Backend: supabase database

## 4. 핵심 요구사항 (Requirements)
- [ ] 페이지는 메인, 어드민 로그인(/admin), 메뉴 관리 페이지(관리자 전용) 이렇게 필요해
- [ ] 메인페이지는 좌측에 메뉴가 따로있을거야. 그 메뉴에는 버튼을 눌러서 필터링을 구현해줘. 필터링은 기본적으로 가격 오름차순으로 구현해줘.
- [ ] 필터링은 카테고리 별로 있을건데 가장 위에 상품 종류 (다발, 바구니, 센터피스, 화병꽂이), 꽃색상 (빨주노초파남보, 흰색, 검은색, 기타), 포장지 색상 (밝은 계열, 어두운 계열, 기타) 이런 필터들을 버튼형식으로 만들어주고 여러개 선택할 수 있도록 해줘.
- [ ] 모든 필터링 버튼을 해제할 수있는 버튼을 메뉴 맨 아래에 구현해줘.
- [ ] 버튼을 누르고 1분 이상 아무 것도 하지 않으면 모든 버튼을 자동으로 해제하는 기능도 구혀해줘.
- [ ] 메인에 리스트 형식으로 필터링된 메뉴들이 나오도록 해줘. 필터링 버튼을 아무것도 누르지 않았다면 메인 섹션에 하나의 이미지가 출력되도록 구현. 리스트는 3단으로 보여줘.
- [ ] `/admin` 경로로 접속했을 때만 로그인 폼 노출
- [ ] 관리자 페이지에서 상품을 추가, 수정, 삭제 할 수 있고, 필터링에 맞게 상품 종류, 꽃색상, 포장지 색상, 가격, 이미지를 입력하여 추가 하도록 구현.
- [ ] 상품을 인기,추천 상품으로 설정할 수 있도록 해줘 추가, 수정할 때 버튼을 누를수 있고 상품 이미지 왼쪽 상단에 추천 혹은 인기 마크가 보이도록해줘

## 5. 제약 사항 (Constraints)
- 별도의 백엔드 서버 없이 supabase만 사용할 것
- 디자인은 최대한 미니멀하고 밝은 베이지톤으로 유지할 것
